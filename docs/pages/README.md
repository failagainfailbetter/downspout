# Downspout Pages source

This directory is the source for the GitHub Pages site.

## Editing plugin text

Each plugin page is a Markdown file under `_products/`.

The short card text comes from the `summary` field in front matter. The longer
body text starts below the front matter and can be expanded manually without
touching the layouts.

## Screenshots

Screenshots live under `assets/plugins/` and are generated by:

```bash
scripts/capture-plugin-screenshots.sh
```

Run that script from the repository root after UI changes, then review the PNGs
before publishing.

## Layout

The shared page structure is in `_layouts/`, and the shared CSS is in
`assets/site.css`.

The GitHub Actions workflow at `.github/workflows/pages.yml` builds this
directory with Jekyll and deploys it to GitHub Pages.

## Local preview

From the repository root:

```bash
jekyll serve --source docs/pages --destination /tmp/downspout-pages-site
```

Then open the URL printed by Jekyll. The configured project-page URL is
`http://127.0.0.1:4000/downspout/` when using the default local port.
